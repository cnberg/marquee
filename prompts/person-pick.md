## 怎么挑

- 用户只说人名（如"成龙的电影"、"看看诺兰"）→ 按候选自然顺序前 {{n}} 部
- 用户带了修饰词（时间 / 题材 / 角色 / 风格等，如"成龙**早期**的电影" / "诺兰的**科幻**片" / "梅丽尔·斯特里普的**喜剧**" / "周润发演的**反派**"）→ 按修饰词过滤，挑最贴合的 {{n}} 部
- 库内符合修饰词的不足 {{n}} 部时，**宁可少给也不要凑数**——把不符合修饰词的作品补进来等于无视用户意图；完全没有匹配的就返空数组（不要勉强）

## 推荐语写什么

聚焦**这部在 {{person_name}} 作品体系中的独特位置**，挑下面任一角度：

- **题材类别**：在该人作品里偏哪类（"{{person_name}} 罕见的喜剧" / "成熟期心理悬疑代表作" / "晚年家庭题材回归"）
- **风格 / 阶段**：早期实验 / 中期巅峰 / 晚期收尾 / 转型作 / 突破之作
- **影史地位**：奥斯卡 / 金棕榈 / 影评人封神（**仅当真有此地位**，否则别瞎扯）
- **故事钩子**：一句话剧情亮点（情节、关系、设定的反差）
- **角色信息**：演员片若 role_detail 给出角色名，可写进去（如"X 饰演冷面杀手肯尼"）

## 写作原则

- **有区分度**：reason 在列表里换给另一部仍然成立，就等于没写。把这部和兄弟作品区分开，比夸它好更重要
- **写电影，不写人**：用户已经知道这是 {{person_name}} 的作品。reason 要告诉读者**这部**有什么独特内容，不是复述身份
- **具体优先**：情节钩子、角色名、影史标签优先；少用形容词（"震撼" / "扣人心弦" / "一气呵成"这类）
